## Compréhension des principes SOLID
Voici ce que j'ai retenu du support de cours :
* **SRP** : Une classe doit avoir une seule responsabilité (une seule raison de changer).
* **OCP** : Le code doit être ouvert à l'extension mais fermé à la modification.
* **LSP** : Les objets d'une classe enfant doivent pouvoir remplacer ceux de la classe parente sans erreur.
* **ISP** : Il vaut mieux plusieurs interfaces spécifiques qu'une seule interface générale.
* **DIP** : Il faut dépendre des abstractions (interfaces) et non des implémentations concrètes.
